# Week 4 — workflow (`phase-3-workflow`)

## Objectives
- Turn smart inbox into an operational execution tool.

## Deliverables
- Case board / list views.
- Status transitions (state machine).
- Assignment rules (category / priority / team).
- Task creation + due dates.
- Overdue warnings + SLA timers.
- Internal notes + activity feed.
- Search + saved filters.

## Acceptance
- Managers can configure assignment rules.
- Operators work entirely from cases/tasks, not email.
- Overdue work is clearly surfaced.
- Stalled work visible immediately.
