# Week 6 — hardening (`phase-5-hardening`)

## Objectives
- Make the product stable for design partners and early paid pilots.

## Deliverables
- E2E tests for email/form → case flow.
- Mobile / responsive QA pass.
- Permissions review.
- Error monitoring + support tooling.
- Pilot onboarding checklist.
- Admin handbook.
- Demo script + sample account.
- Launch checklist + issue triage process.

## Acceptance
- No broken core flows in demo.
- First pilot can be onboarded with documented steps.
- Error paths visible and recoverable.
- Team can run outbound demos confidently.
